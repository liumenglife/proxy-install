# proxy-ui 局部重组与状态同步设计

## 目标

对 `proxy-ui` 做局部 UI/UX 重组和状态逻辑修正，解决按钮位置混乱、区域层级不明显、速度噪音、选中态与后台不同步、节点摘要信息可读性差等问题。

本次不重做整页架构，只在现有 vanilla HTML/CSS/JS 基础上重组顶部控制区、自动组区域、手动选择区域、分组摘要行和节点卡片。

## 范围

包含：

- 顶部按钮区重组。
- 自动组代理选择器区域强化。
- 新增手动代理选择区域标题。
- `全部聚合`、`按机场`、`按地区` 子区域视觉强化。
- 全局 `定位当前节点` 移除，改为每个具体分组内右侧操作。
- `全部展开` / `全部收起` 合并为单一状态切换控件。
- 速度指标从 UI 中移除。
- 分组摘要行删除显式标签，通过布局、颜色、图标表达语义。
- 手动/自动选中态互斥与后台状态同步。
- 地区/国家显示对应旗帜或区旗。

不包含：

- TUN 二阶段能力。
- 新增真实下载速度测试。
- 改造 sing-box/sub-store 后端生成逻辑。

## 顶部控制区

`代理控制台` 标题右侧应放置顶部主操作组：

- `立即同步`
- `全部测速`
- `全部展开` / `全部收起` 单一切换控件

原 `.toolbar` 独立工具条删除。`全部测速` 不再放在手动区域上方的孤立工具条中。

`全部展开` / `全部收起` 必须是同一个按钮或链接的两种状态，不能是两个控件。规则：

- 如果不是所有手动分组都展开，控件显示 `全部展开`。
- 点击后展开全部手动分组，并把同一个控件文案切换为 `全部收起`。
- 如果所有手动分组都已展开，控件显示 `全部收起`。
- 点击后收起全部手动分组，并把同一个控件文案切换为 `全部展开`。
- 用户手动展开或收起某个分组后，也要重新计算控件文案。
- DOM 中不得同时存在 `expand-all-btn` 和 `collapse-all-btn` 两个可见控件。

## 自动组代理选择器区域

`自动组代理选择器` 是一级能力区域，不再是普通小号 `label`。

视觉要求：

- 使用大字号、加粗、图标、区域卡片或标题底纹。
- 与 `手动代理选择区域` 同级、同字号、同字重、同图标规格、同微特效。
- 图标语义可以是闪电、自动化、雷达或类似矢量图标。
- 下拉框是该区域的主要控件，视觉上必须归属于自动组区域。

## 当前模式状态提示

`当前为自动组`、`当前为手动组：按地区/台湾/手动组` 不能再作为容易忽略的小号 `hint` 文本。

应改为醒目的状态横幅或状态胶囊卡片：

- 文案示例：`当前模式：自动组`。
- 文案示例：`当前模式：手动组 · 按地区 / 台湾`。
- 自动组状态使用蓝/紫系视觉。
- 手动组状态使用绿/琥珀系视觉。
- 必须带图标和背景容器。
- 字号至少接近区域副标题级别，不能低于普通说明文字。
- 状态提示应与自动/手动区域有明确视觉关联。

## 实际路由标签

`实际路由标签` 继续保留，并且是唯一展示真实最终路由的区域。

无论当前选择自动组还是手动组，它都必须显示：

```text
代理选择标签当前值 → 实际出站节点
```

示例：

```text
按地区/香港/自动组 → 香港-猫熊机场-🇭🇰 直连-福利-V114-香港-0.1x-仅限emby
```

实际路由标签在视觉上必须拆成三段语义，而不是一整行难读文本：

- 第一段：自动/手动分类，例如 `按地区 / 美国 / 自动组`。
- 第二段：所属机场信息，例如 `美国 · 猫熊机场`。
- 第三段：具体节点信息，例如 `🇺🇸 直连-V350-美国-1x-NF&HBO&Disney*`。

三段之间可以使用箭头、细线、分隔符或连线轨道，但必须语义清晰、视觉不违和。

`frontend-design` 定稿方向：实际路由标签采用“航线轨道”式结构，三段都是 compact chip，但权重不同。

- 分类 chip 使用冷色低饱和背景，表达当前控制入口。
- 机场 chip 使用中性深色背景和细边框，表达归属来源。
- 节点 chip 使用最长、最亮的内容槽，允许省略，表达最终出口。
- 三段之间用细箭头或细连接线串联，像代理流量从分类、机场到具体节点的路径。
- 不使用大面积渐变或夸张装饰，避免抢夺顶部控制区注意力。

示意：

```text
[按地区 / 美国 / 自动组]  →  [美国 · 猫熊机场]  →  [🇺🇸 直连-V350-美国-1x-NF&HBO&Disney*]
```

如果节点名称无法可靠拆出第三段，应保守回退为第二段空缺、第三段显示完整节点名，不能错误拆分。

拆分规则：

- 节点名采用既有 `{地区}-{机场}-{节点名}` 约定解析。
- 第一个 `-` 之前为地区。
- 第二个 `-` 之前为机场。
- 第二个 `-` 之后为具体节点信息。
- 机场 chip 展示为 `{地区} · {机场}`，其中地区可带旗帜或区旗。
- 具体节点 chip 展示第三段原文，不重复前面的地区和机场。

## 手动代理选择区域

在手动分组列表上方新增一级标题：`手动代理选择区域`。

视觉要求：

- 与 `自动组代理选择器` 同级。
- 使用同字号、同字重、同图标规格、同微特效。
- 表达自动选择与手动选择是并列的两种能力。
- 不能只是普通文本。

## 手动区域子分类

`全部聚合`、`按机场`、`按地区` 是 `手动代理选择区域` 的子级。

视觉要求：

- 三者使用统一子区标题样式。
- 字号小于 `自动组代理选择器` 和 `手动代理选择区域`。
- 不能只是光秃秃文字。
- 每个标题必须带矢量图标：
  - `全部聚合`：网格、聚合或层叠图标。
  - `按机场`：服务器、机房或节点组图标。
  - `按地区`：地图定位图标。
- 可加入图标胶囊、标题底纹、细边框或轻微 hover 光晕，但不能喧宾夺主。

## 地区/国家旗帜

具体地区或国家名称需要带对应旗帜或区旗。

示例：

- `🇭🇰 香港`
- `🇲🇴 澳门`
- `🇸🇬 新加坡`
- `🇺🇸 美国`
- `🇯🇵 日本`
- `🇹🇼 台湾`
- `🇬🇧 英国`
- `🇫🇷 法国`
- `🇩🇪 德国`
- `🇨🇦 加拿大`
- `🇦🇺 澳大利亚`

规则：

- 旗帜用于地区/国家分组标题和具体地区分组摘要名称。
- 未知地区不得猜测错误旗帜，回退为通用地图图标。
- 机场名称不强制加旗帜。
- 地区名称不能只是光秃秃文字，必须做成有视觉设计感的地域徽章或地域标题单元。
- 地域徽章至少包含：旗帜或区旗、地区名称、低饱和背景、细边框或内阴影、轻微 hover/focus 微特效。
- `香港`、`澳门`、`新加坡`、`美国` 等具体地区在 `按地区` 子区下必须显示为类似 `🇭🇰 香港` 的视觉单元，而不是裸文本。
- 地域徽章应与节点健康指标、当前节点摘要形成清晰层级：地区名称是入口标题，健康指标是状态，节点名是描述。
- 地域徽章不能过度装饰，不能压过节点信息和右侧操作组。

推荐形态：

```text
[ 🇭🇰 香港 ]    12 / 20
               238ms      香港-猫熊机场-🇭🇰 直连-福利-V114...        [测速][定位📌]
```

`按地区` 子区标题自身也需要地图定位类矢量图标，与具体地域徽章形成父子视觉关系。

## 具体分组摘要行

每个具体分组摘要行使用左右布局。

左侧：

- 分组名称。
- 健康指标组。
- 当前节点摘要。

右侧：

- 操作组容器，包含 `测速` 和 `定位📌`。

桌面端右侧操作组必须整体右对齐，不能跟在节点文字后面混排。移动端可以换行，但仍要靠右，或占满宽度后内部右对齐。

右侧操作组必须有独立背景容器，例如 `.summary-actions`。

`定位📌` 只出现在具体分组行内，不存在全局 `定位当前节点`。

## 分组摘要信息表达

摘要行不得显示显式标签：

- 不显示 `当前：`。
- 不显示 `延时：`。
- 不显示 `可用节点数：`。
- 不显示 `速度：`。

摘要行只显示值本身，通过位置、图标、样式、颜色表达语义。

推荐结构：

```text
🇭🇰 香港    12 / 20
           238ms      香港-猫熊机场-🇭🇰 直连-福利-V114...        [测速][定位📌]
```

健康指标组规则：

- `12 / 20` 和 `238ms` 必须相邻，不能被节点名称隔开。
- `12 / 20` 表示可用节点数与总节点数。
- `12` 与 `20` 必须视觉区分：
  - `12` 使用更亮颜色和更高字重。
  - `/` 与 `20` 使用低饱和灰色和较低权重。
- `238ms` 表示当前分组所选节点延时。
- `12 / 20` 与 `238ms` 组成 compact health metric group。
- 两个指标采用轻微错位的双层排版，不使用 `<sup>`，不把字体缩小到影响可读性。
- 节点名称从健康指标组右侧开始，过长时单行省略，`title` 可显示完整值。

延时状态不显示未知文案。没有有效 `ms` 的情况一律按超时处理，包括 timeout、接口无响应、接口错误、历史为空或无法解析。UI 显示 `timeout`，不得显示 `--`、`unknown` 或 `延时：--`。

## 节点卡片

所有具体节点移除速度 chip。

节点卡片结构：

- 节点名称作为主内容。
- 延时数据放在卡片右下角。
- 延时用小正方形或接近正方形的 compact badge 包裹。
- badge 有 hover/focus 微动效。
- 不显示速度，不显示 `--` 速度占位。

延时颜色语义：

- excellent `<500ms`：绿色。
- good `500-999ms`：浅绿或青绿。
- warning `1000-1999ms`：明确黄色/琥珀色，不能偏棕黑到看不出黄色。
- poor `>=2000ms`：紫色。
- timeout：红色。
- 没有有效 `ms`、接口无响应、接口错误或历史为空：全部归入 timeout，红色。

## 速度指标移除

UI 中完全移除速度指标。

要求：

- 节点卡片不显示速度 chip。
- 分组摘要不显示 `速度：--`。
- 没有速度值时，连 `速度` 两个字都不能出现。
- 保留 `测速` 功能，但它的语义是延时测试。

## 选中态同步逻辑

选中态必须与 `代理选择标签` 后台状态一致，不能显示噪音选中数据。

规则：

- 如果当前选择的是自动组，手动代理选择区域禁止显示任何节点为选中态。
- 如果当前选择的是自动组，只有 `实际路由标签` 显示该自动组解析出的实际节点。
- 如果当前选择的是手动组，自动组下拉不显示为当前有效选择。
- 如果当前选择的是手动组，只允许 `代理选择标签` 指向的那个手动组显示选中节点。
- 如果选择了 A 机场节点，只有 `按机场/A机场/手动组` 可显示该节点选中，其他机场、所有地区、全部聚合都不能显示选中态。
- 如果选择了 A 地区节点，只有 `按地区/A地区/手动组` 可显示该节点选中，其他地区、所有机场、全部聚合都不能显示选中态。
- 如果选择了 `全部聚合/手动组` 的节点，只有 `全部聚合/手动组` 可显示选中态，机场和地区分组不能因为节点名相同而显示选中态。

## 键盘快捷键

新增两个键盘快捷键：

- `Command+K`：展开全部手动分组。
- `Command+L`：收起全部手动分组。

规则：

- 不区分大小写，`Command+k` 与 `Command+K` 等价。
- macOS 使用 `metaKey` 识别 Command。
- 不拦截输入框、下拉框、文本域内的输入行为。
- 触发后需要同步更新顶部单一展开/收起控件文案。
- 可在状态横幅或状态提示中短暂提示动作结果，例如 `已展开全部节点`、`已收起全部节点`。

## 验收标准

UI 验收：

- 顶部标题右侧存在 `立即同步`、`全部测速`、单一展开/收起切换控件。
- 不存在全局 `定位当前节点`。
- 不存在独立旧 `.toolbar` 工具条。
- `自动组代理选择器` 是一级视觉区。
- `手动代理选择区域` 是一级视觉区。
- 两个一级视觉区同级、同字号、同微特效。
- 当前模式状态提示是醒目的横幅或胶囊卡片，不是小号 hint。
- `全部聚合`、`按机场`、`按地区` 有统一子区标题样式和矢量图标。
- 地区/国家分组显示正确旗帜或区旗，未知地区回退通用图标。
- 地区/国家名称不能裸文本显示，必须渲染为带旗帜、背景、边框或内阴影的地域徽章。
- 每个具体分组摘要右侧有 `.summary-actions`，包含 `测速` 和 `定位📌`。
- `测速` 和 `定位📌` 右对齐，不与节点文字混排。
- 页面中不得出现 `当前：`、`延时：`、`可用节点数：`、`速度：`。
- `12 / 20` 中可用数与总数视觉区分。
- `12 / 20` 和 `238ms` 相邻且采用轻微错位双层排版。
- 节点卡片不显示速度 chip，也不显示速度 `--`。
- 节点延时在卡片右下角 compact badge 中显示。
- warning 延时背景必须明确是黄色/琥珀系。
- 延时无有效 `ms` 时必须显示 timeout，不得显示 unknown 或 `--`。
- 实际路由标签必须拆成分类、机场、具体节点三段式视觉结构。
- `Command+K` 展开全部手动分组，`Command+L` 收起全部手动分组，不区分大小写。

逻辑验收：

- 自动组模式下，手动区域无任何选中节点。
- 手动机场组选择时，只对应机场组显示选中节点。
- 手动地区组选择时，只对应地区组显示选中节点。
- `全部聚合/手动组` 选择时，只 `全部聚合` 显示选中节点。
- `实际路由标签` 始终显示真实最终节点。
- 单一展开/收起控件能随全局点击和局部分组展开状态正确切换文案。

测试验收：

- 单元测试覆盖健康指标渲染、速度移除、选中态互斥、旗帜映射、单一展开/收起状态机。
- E2E 覆盖顶部按钮位置、全局定位按钮移除、分组右侧操作组、区域标题强化、状态横幅、节点无速度、自动/手动选中态互斥。
- `bun test` 通过。
- `bunx playwright test --reporter=line` 通过。
- `docker compose up -d --build proxy-ui` 后 9091 页面验证通过。
